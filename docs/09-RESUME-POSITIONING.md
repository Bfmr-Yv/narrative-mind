# 简历定位建议

> 本文档为项目作者提供不同岗位方向下的项目描述策略。

---

## 核心定位

**不要写"小说写作工具"**，要写：

> **多 Agent 协作的复杂知识管理系统**

---

## 不同岗位的描述策略

### 1. AI 应用开发 / ML 工程师

**项目名称**：Narrative Mind — 多 Agent 叙事一致性系统

**描述**：
- 设计并实现 8 引擎多 Agent 协作架构，支持事件驱动调度、并行执行和三级降级熔断
- 构建 SQLite + LanceDB 混合存储方案，实现文本、向量嵌入、图结构、时序数据的统一管理
- 开发 Prompt Registry 与 Tier 0-4 成本预算层，实现多模型动态路由与分级定价，月度成本控制在 $20 以内
- 部署本地 Qwen2.5-7B 模型处理 70% 常规任务，结合云端 MiMo/DeepSeek 处理深度推理
- 基于心理学 PAD 情感模型建立角色行为状态机，实现基于语料锚定的行为一致性检测
- 技术栈：Python 3.11、React + TypeScript、LanceDB、SQLite、Jinja2

**关键量化指标**：
- 8 引擎协同工作，Phase 1 响应延迟 < 2s
- 本地模型覆盖 70% 调用量，云端成本 $20/月硬顶
- 语料切片 500 字场景级，检索相似度阈值 0.3

---

### 2. 后端 / 系统架构工程师

**项目名称**：Narrative Mind — 事件驱动型多模块协作系统

**描述**：
- 设计编排器-引擎-守卫三层架构，实现模块间的契约式解耦和统一记忆写入
- 实现事件驱动的引擎调度系统，按用户操作类型动态路由到相关引擎，避免全量串行调用
- 构建三层记忆模型（工作记忆/情节记忆/永久记忆），实现会话级到跨书级的数据生命周期管理
- 设计八维一致性守卫系统，支持三级警报（INFO/WARN/CRITICAL）和冲突自动仲裁
- 实现 Tier 0-4 分级降级策略，三层熔断机制保障系统在预算耗尽时平滑降级
- 技术栈：Python 3.11、SQLite、事件驱动架构、契约优先设计

**关键量化指标**：
- 模块间零直接依赖，所有通信通过编排器路由
- 一致性守卫 Phase 1 检查 3 个维度，Phase 4 扩展至 8 个
- 降级策略支持 3 层熔断（50%/80%/95% 预算阈值）

### 3. 全栈工程师

**项目名称**：Narrative Mind — 本地优先的 AI 增强型桌面应用

**描述**：
- 独立设计并开发完整的本地优先桌面应用（Python 后端 + React 前端）
- 前端实现章节编辑器、侧边栏建议卡片、待办清单面板、冲突对话弹窗
- 后端实现多 Agent 协作架构，支持并行引擎调度和统一记忆管理
- 集成本地向量数据库 LanceDB 和本地 LLM Qwen2.5-7B，实现纯离线基础功能
- 设计意图标记协议（`<!-- INTENT: xxx -->`），实现作者与 AI 系统之间的元通信
- 全栈独立开发，项目规模 15-25 万行，使用 Git 版本控制

**关键量化指标**：
- MVP 3-5 万行代码，8-12 周可交付
- 桌面端纯本地运行，零外部依赖基础模式
- 支持 Markdown 导入/导出，不锁定用户数据

## 与竞品的差异化定位

| 竞品 | 定位 | Narrative Mind 的差异 |
|------|------|----------------------|
| Sudowrite | AI 生成文本 | 我们**不生成**文本，我们**管理复杂性** |
| NovelAI | AI 续写 | 我们是作者的参谋，不是代笔 |
| Scrivener | 项目管理 | 我们加了 AI 引擎层，主动检测矛盾 |
| Grammarly | 语法检查 | 我们检查叙事逻辑，不是语法 |

**一句话区分**：别人让 AI 替你写，我们让 AI 帮你想。

---

## 面试可讲的亮点

1. **架构复杂度**：8 引擎 + 编排器 + 守卫 + 三层记忆，从零设计的分布式思想
2. **成本工程**：Tier 0-4 分级定价 + 三层熔断，在 $20/月约束下的极端优化
3. **理论到工程**：将文学理论（PAD 情感模型、叙事学）形式化为可计算的算法
4. **人类优先**：AI 只给建议不做决策的设计哲学，意图标记协议
5. **独立性**：全程独立设计、开发、文档化，无需外部协作
