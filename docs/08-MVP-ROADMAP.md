# MVP 路线图

> 目标：从文档设计到可运行的单章分析器原型，分 4 个 Phase 逐步构建完整的 Narrative OS。

---

```

Week 1-12             Week 13-24             Week 25-36             Week 37+
══════════            ════════════           ════════════           ═══════════
```

## Phase 1（当前 | Week 1-12）：可跑通的单章分析器

### 目标
> 输入 1 章 Markdown + 角色历史 2-3 章 → 输出 JSON 报告（行为模式对比 + 设定一致性 + 建议）

### 交付物

| 模块 | 交付内容 | 验证标准 |
|------|---------|---------|
| **语料锚定** | 20 切片（红楼 10 + 自有 10）+ LanceDB 向量库 | 输入场景文本，能返回 top-3 相似切片 |
| **角色引擎** | PAD 情感追踪 + 基础行为预测 | 行为预测与语料模式的相关性 > 0.6 |
| **世界引擎** | 修仙力量体系校验 + 空间规则 | 能检测到明显的规则冲突 |
| **级联修正器** | 关键词扫描 + 待办清单生成 | 修改文本后能列出受影响的章节 |
| **一致性守卫** | 3 维检查 + WARN/CRITICAL 警报 | 交叉验证两引擎输出，发现冲突时阻断 |
| **记忆系统** | 工作记忆 + 简化情节记忆（SQLite） | 会话内能查询历史 PAD 状态和行为日志 |
| **编排器** | 事件路由 + 引擎调度 + 记忆统一写入 | 正确路由 5 种 UserAction |
| **前端** | 章节编辑器 + 侧边栏建议卡片 | 能粘贴文本、触发分析、查看结果卡片 |
| **成本层** | 月度配额追踪 + 超量停止 | 达到 $20 时停止 API 调用 |

### Phase 1 排除项
- ❌ 叙事/文辞/主题/经济/读者预期/构思引擎
- ❌ 永久记忆
- ❌ 级联修正器的因果推理
- ❌ 意图标记系统
- ❌ 章节级快照
- ❌ 复杂 Dashboard
- ❌ 修仙 Profile（仅用通用 Profile）

---

## Phase 2（Week 13-24）：核心引擎扩展

### 新增
- **叙事引擎**：伏笔追踪 + 因果图谱
- **文辞引擎**：风格一致性检测
- **完整一致性守卫**：8 维激活
- **完整级联修正器**：因果推理 + 影响传播
- **三层记忆**：永久记忆层 + 冷热分层
- **前端 MVP**：大纲编辑器 + 可视化面板

### 验证标准
> 对 10 章以上的连载作品进行全书审查，能检测到跨章节的伏笔未回收和因果矛盾。

---

## Phase 3（Week 25-36）：完整引擎套件

### 新增
- **主题引擎**：主题提取与演变追踪
- **经济引擎**：资源系统一致性校验
- **读者预期引擎**：信息差建模
- **构思引擎**：意象体系管理
- **作者指纹 v1**：风格基线提取
- **反馈学习层**：从用户决策中学习

### 验证标准
> 完整 8 引擎协同工作，对修仙类长篇网文的全书审查能在 30 分钟内完成（Tier 4 异步）。

---

## Phase 4（Week 37+）：生态打磨

### 新增
- **完整版本控制**：Git-like 分支 + 合并 + A/B 测试
- **可视化图谱**：因果图、角色关系图、主题演变图
- **类型化动态生成**：从语料自动生成 GenreProfile
- **社区语料共享**：公共域语料库扩展到 100+ 切片
- **多类型支持**：玄幻、都市、科幻等类型 Profile

## 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 语料切片质量不足，检索结果无意义 | 中 | 高 | 人工精校验每片，只审不标 |
| 本地 7B 模型性能不够 | 中 | 中 | 预留 $5/月云端预算处理降级任务 |
| 单章分析报告无价值（废话） | 低 | 高 | 与真实网文作者验证报告质量 |
| 开发周期超预期 | 高 | 中 | Phase 1 已经砍到最小可验证单元 |
| API 成本超预算 | 低 | 高 | 三层熔断机制自动触发 |
