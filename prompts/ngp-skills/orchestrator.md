# NGP 编排器 Skill

> **Phase**: 1  
> **状态**: 活跃  
> **依赖**: 角色引擎、世界引擎、一致性守卫

## 职责

接收用户操作，路由到相应引擎，协调引擎返回结果，统一写入记忆。

## 任务路由表

| 任务类型 | 触发条件 | 路由引擎 | 输出 |
|----------|----------|----------|------|
| 文档修改 | 修改 .md 文件 | 命名守卫 + 重复检测 | 守卫报告 |
| 代码生成 | 创建/修改 .py 文件 | Phase 边界检查 | 边界报告 |
| 架构讨论 | 涉及引擎设计 | 命名冻结检查 | 命名报告 |
| 命名统一 | 检测废弃名称 | 命名守卫 | 违规报告 |
| 成本调整 | 涉及成本数字 | 成本校验器 | 一致性报告 |
| Phase越界预警 | 引用 Phase 2+ 引擎 | Phase 边界检查 | 阻断报告 |

## 四层门禁检查清单

### L1: 记忆层（写入前）

- [ ] 检查连续重复行（`uniq -d`）
- [ ] 日期格式为 YYYY-MM-DD
- [ ] 成本数字与 `04-COST-BUDGET.md` 一致

### L2: 扩展层（模块引用）

- [ ] 命名与 `02-CONTRACTS.md` 对齐
- [ ] 无废弃名称（Style Engine → Prose Engine，Impact Radar → Cascade Fixer）
- [ ] 引用的引擎在当前 Phase 激活

### L3: 集成层（跨模块）

- [ ] Phase 标记正确（Phase 1 不引用 Phase 2+ 接口）
- [ ] 一致性守卫维度匹配（Phase 1 仅 3 维）
- [ ] 成本数字自洽（与 Tier 表交叉核对）

### L4: 编程层（代码实现）

- [ ] 仅使用 Python 标准库
- [ ] 无外部依赖引入
- [ ] 代码风格与现有模块一致

## 引用文档

- 一致性守卫：`06-CONSISTENCY-GUARDIAN.md`
- Agent 约束：`10-AGENT-CONSTRAINTS.md`
- 命名冻结表：`10-AGENT-CONSTRAINTS.md` §2
- Phase 边界：`08-MVP-ROADMAP.md` §Phase 1 排除项

## 执行流程

1. 接收用户操作
2. 根据任务类型路由到对应检查
3. 执行四层门禁检查
4. 汇总检查结果
5. 通过 → 执行操作；不通过 → 返回报告

## 降级策略

- 单检查超时 30s → 跳过该检查，标记为 WARN
- 多检查失败 → 优先返回 CRITICAL 级别问题
